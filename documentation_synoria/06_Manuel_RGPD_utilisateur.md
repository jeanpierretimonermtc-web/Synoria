# Manuel RGPD utilisateur - Synoria

Version : 1.5.0  
Public : praticien utilisateur  
Important : ce document est une aide operationnelle, pas un avis juridique.

## Principe

En utilisant Synoria pour gerer des dossiers patients, le praticien traite des donnees personnelles et potentiellement des donnees de sante ou donnees sensibles.

Le praticien reste responsable de ce qu'il collecte, saisit, conserve, exporte, transmet et supprime.

Synoria fournit des outils d'aide :

- consentement patient ;
- notice patient ;
- journal d'acces ;
- alertes de conservation ;
- registre des traitements article 30 ;
- sauvegardes chiffrees selon configuration ;
- stockage local par defaut.

Ces outils ne dispensent pas le praticien de ses obligations professionnelles.

## Engagement du praticien

En utilisant Synoria, le praticien s'engage a :

- collecter uniquement les informations necessaires a son activite ;
- informer les patients de l'utilisation de leurs donnees ;
- documenter le consentement lorsque necessaire ;
- proteger l'acces au logiciel ;
- ne pas partager son mot de passe ;
- effectuer des sauvegardes regulieres ;
- conserver les sauvegardes dans un endroit securise ;
- ne pas inscrire de donnees sensibles dans Google Calendar ;
- ne transmettre un dossier qu'avec une base legitime ou l'accord du patient ;
- respecter les demandes d'acces, de rectification ou de suppression lorsque la loi le permet ;
- supprimer ou archiver les donnees lorsqu'elles ne sont plus necessaires ;
- signaler les incidents importants selon les obligations applicables.

## Donnees traitees

Synoria peut contenir :

- identite du patient ;
- coordonnees ;
- date de naissance ;
- profession ;
- medecin traitant ;
- antecedents ;
- traitements en cours ;
- notes d'anamnese ;
- observations ;
- analyses ;
- traitements effectues ;
- conseils ;
- rendez-vous ;
- factures ;
- exports ;
- consentement et date de consentement.

Ces donnees doivent etre considerees comme confidentielles.

## Information du patient

Le patient doit comprendre :

- qui traite ses donnees ;
- pourquoi les donnees sont collectees ;
- quelles categories de donnees sont conservees ;
- combien de temps elles sont conservees ;
- qui peut y acceder ;
- comment exercer ses droits ;
- comment contacter le praticien ;
- comment contacter la CNIL en cas de reclamation.

Synoria peut aider a generer ou conserver une notice patient. Le texte doit etre adapte a la pratique reelle.

## Consentement

Le consentement n'est pas toujours la seule base legale possible, mais lorsqu'il est utilise, il doit etre libre, specifique, informe et non ambigu.

Bonne pratique :

1. Expliquer au patient l'objectif du dossier.
2. Expliquer les donnees conservees.
3. Expliquer la duree de conservation.
4. Enregistrer le consentement dans Synoria.
5. Mettre a jour si les finalites changent.

Pour les donnees sensibles, un consentement explicite ou une autre base applicable doit etre analyse selon la situation professionnelle.

## Droits des patients

Les patients peuvent demander, selon les cas :

- l'acces a leurs donnees ;
- la rectification ;
- l'effacement ;
- la limitation du traitement ;
- la portabilite ;
- l'opposition ;
- des informations sur le traitement.

Le praticien doit repondre dans un delai raisonnable et conforme aux obligations applicables. En pratique, garder une trace de chaque demande et de la reponse apportee.

## Registre des traitements

Synoria peut exporter un registre des traitements.

Le registre doit decrire :

- le responsable du traitement ;
- les finalites ;
- les categories de personnes ;
- les categories de donnees ;
- les destinataires ;
- la duree de conservation ;
- les mesures de securite ;
- les transferts eventuels hors UE.

Le registre doit etre conserve et mis a jour.

## Journal d'acces

Le journal d'acces permet de tracer certaines consultations ou actions dans les dossiers.

Utilisation :

- verifier les acces recents ;
- documenter une intervention ;
- appuyer un diagnostic ;
- detecter une anomalie.

Le journal n'est utile que si l'acces au poste et au logiciel est protege.

## Duree de conservation

La duree de conservation doit etre adaptee a la profession, aux obligations applicables et aux finalites reelles.

Synoria permet de configurer une duree et d'afficher des alertes. Le praticien doit choisir une duree coherente et documentee.

Recommandation : demander conseil a son organisation professionnelle, son assureur, son juriste ou son DPO si disponible.

## Sauvegardes

Les sauvegardes contiennent des donnees sensibles.

Regles :

- les proteger comme le logiciel principal ;
- eviter les supports non securises ;
- ne pas les envoyer par email non chiffre ;
- tester regulierement la restauration ;
- supprimer les anciennes sauvegardes devenues inutiles ;
- conserver le mot de passe ou la cle separement.

## Google Calendar et RGPD

Google Calendar ne doit pas contenir de notes cliniques.

Bon format d'evenement :

```text
Consultation
```

Mauvais formats :

```text
Dupont - douleurs lombaires
Consultation anxiete Mme Martin
Seance MTC infertilite
```

Le calendrier externe doit servir au planning, pas au dossier patient.

## Export et transmission

Avant d'exporter ou transmettre :

1. Verifier le destinataire.
2. Verifier le contenu du document.
3. Minimiser les donnees.
4. Obtenir l'accord du patient si necessaire.
5. Utiliser un canal securise.
6. Conserver une trace de l'envoi si utile.

## Incident de securite

Exemples :

- ordinateur vole ;
- cle USB perdue ;
- sauvegarde envoyee au mauvais destinataire ;
- mot de passe divulgue ;
- donnees patients exposees ;
- acces non autorise.

Procedure :

1. Identifier les donnees concernees.
2. Evaluer le risque pour les patients.
3. Changer les mots de passe.
4. Revoquer les acces externes si necessaire.
5. Restaurer depuis une sauvegarde saine si besoin.
6. Documenter l'incident.
7. Verifier si une notification CNIL ou patient est necessaire.
