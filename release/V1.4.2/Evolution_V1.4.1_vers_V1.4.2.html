⚠ Migration base de données
Cette version applique automatiquement une migration au premier lancement :
  • Migration v14 — Création de la table session_templates pour la sauvegarde des modèles de séance
Aucune action requise — la migration est transparente et non destructive.
1
Templates de séance
NOUVEAU Sauvegarde et chargement de modèles cliniques
  • Bouton ⭐ Sauver comme modèle en haut de la page Nouvelle Séance — capture tous les champs cliniques remplis
  • Bouton 📂 Modèles pour charger un modèle existant — les données saisies sont restaurées instantanément
  • Chaque modèle possède un nom et une description optionnelle
  • Bouton Supprimer par modèle avec confirmation
  • Les modèles sont persistants dans la base de données SQLite (table session_templates)
  • Champs capturés : motif, évolution, tags, anamnèse, observation, traitement, réactions, techniques, langue, pouls, constitution, type de corps, teint, diagnostic MTC, 5 éléments, causes, analyse, principes, points, oreille, plantes, barrage (niveaux 1–4)
2
Recherche globale
NOUVEAU Barre de recherche universelle — Ctrl+K
  • Raccourci Ctrl+K (ou ⌘K sur Mac) pour ouvrir la palette de recherche
  • Recherche simultanée dans les patients et les séances
  • Résultats en temps réel avec affichage du nom, de la date et du motif
  • Navigation rapide : clic ou Entrée pour accéder au dossier ou à la séance
  • Fermeture avec Échap ou clic hors du panneau

Paramètres — nouvelles fonctions
NOUVEAU Badge de version installée
  • L'onglet Sécurité des paramètres affiche désormais la version exacte installée
  • Badge vert À jour ou orange Mise à jour prête selon l'état de la mise à jour
  • Affiché en grands caractères monochromes pour un repérage immédiat
NOUVEAU Vérification d'intégrité des sauvegardes
  • Nouveau bouton 🔍 Vérifier une sauvegarde dans l'onglet Sauvegardes
  • Ouvre un sélecteur de fichier pointant sur un fichier .json.enc
  • Déchiffre et valide la structure du fichier — confirme que la sauvegarde est lisible et non corrompue
  • Affiche le nombre de patients et de séances contenus dans la sauvegarde vérifiée

Fiabilité et corrections
CORRECTION Page blanche sur « Nouvelle Séance »
  • Correction d'une erreur JavaScript Temporal Dead Zone : loadTemplates était référencé avant sa déclaration const
  • La page Nouvelle Séance s'ouvre désormais correctement sans page blanche
CORRECTION ErrorBoundary — affichage des erreurs React
  • Un composant ErrorBoundary entoure toutes les pages de l'application
  • En cas d'erreur de rendu, le message d'erreur et la stack trace s'affichent à la place de la page blanche
  • Bouton ↺ Réessayer pour relancer le rendu sans redémarrer
CORRECTION Erreurs de cache GPU dans la console
  • Masquage des erreurs cache_util_win.cc / gpu_disk_cache.cc causées par les permissions Windows sur le cache Chromium
  • Le flag disable-gpu-shader-disk-cache est activé au démarrage d'Electron
  • La console de développement est désormais propre au lancement

Mise à jour depuis la version 1.4.1
Windows — Installateur
  • Lancez Synoria Setup 1.4.2.exe — l'installateur met à jour l'application automatiquement
  • Vos données, paramètres et mots de passe sont intégralement conservés
Windows — Clé USB (portable)
  • Remplacez Synoria 1.4.1.exe par Synoria 1.4.2.exe sur votre clé USB
  • Le dossier data/ est conservé — vos données sont intactes
  • La migration v14 s'applique automatiquement au premier lancement
macOS
  • Téléchargez le nouveau DMG depuis GitHub Actions (voir Mac/README_Mac.txt)
  • Glissez la nouvelle version de Synoria.app dans le dossier Applications (remplace l'ancienne)
  • Vos données sont stockées dans ~/Library/Application Support/Synoria/ et ne sont pas affectées
