📅 Calendrier & Planning
🔒
Nouveau
Blocs personnels / Indisponibilités
Bouton ⊘ Perso / Indispo dans la toolbar du calendrier. Permet de bloquer un créneau horaire ou une journée entière avec un motif libre (formation, congé, RDV perso…). Affichage distinct avec hachures violettes, bordure pointillée et badge ⊘ dans les en-têtes de colonnes.
🔄
Nouveau
Synchronisation bidirectionnelle Google Calendar
Bouton ⟳ Sync GCal dans le calendrier (visible uniquement si GCal connecté). Importe les événements Google Calendar vers Synoria sur la période visible. Les RDV Synoria continuent d'être poussés automatiquement vers Google Calendar. Backfill automatique au chargement du calendrier.
🎨
UX
Meilleure différenciation visuelle des RDV
Les RDV patients ont maintenant une barre accent gauche de 4 px colorée selon leur statut (bleu = planifié, vert = réalisé, orange = en attente). Les blocs perso ont une texture hachurée distincte. Impossible à confondre d'un coup d'œil.
🗓
Correction
Navigation directe après enregistrement d'une séance
Après avoir enregistré une séance avec un prochain RDV, le calendrier s'ouvre automatiquement sur la date du RDV créé — plus besoin de chercher le bon mois.
📋 Nouvelle Séance
📋
Nouveau
Accordéon « Séance précédente »
Dès qu'un patient est sélectionné, un accordéon replié apparaît entre « Informations patient » et « Motif ». Un clic déplie le résumé complet de la dernière séance : motif, anamnèse, évolution, diagnostic MTC, traitement, barrage, conseils, et note laissée pour cette séance. Fonctionne pour tous les modes (MTC, plugins tiers, mode simple).
🔗
Correction
Synchronisation prochain RDV → Calendrier
Correction d'un bug critique : le RDV était créé après la sauvegarde de la séance, donc son ID n'était jamais persisté. Résultat : chaque enregistrement créait un doublon. Désormais, le RDV est créé en premier, son ID est inclus dans le payload de la séance.

💰 Comptabilité
📊
Nouveau
Bouton « 📊 Compta » sur chaque séance
Sur la page Séances, chaque séance affiche un bouton 📊 Compta à côté de « ✏️ Modifier ». Un clic ouvre une modal permettant de choisir le type de consultation (avec son prix) et d'incrémenter la ligne correspondante dans le tableau mensuel. L'opération est optionnelle et peut être faite à tout moment, même sur des séances passées.
🔐 Sécurité
🔑
Sécurité
Mot de passe admin : suppression du plaintext en source
Le mot de passe d'accès au panel admin était codé en clair dans le source ('Synoria@Dev2026') avec un hash SHA-256 sans sel. Remplacé par une génération aléatoire au premier lancement (128 bits d'entropie, PBKDF2 100 000 itérations). Le mot de passe apparaît une seule fois dans synoria.log avec le tag [ADMIN_SETUP].
📦
RGPD
Polices chargées localement (suppression Google Fonts)
Les polices DM Sans et DM Serif Display étaient chargées depuis fonts.googleapis.com à chaque démarrage, envoyant l'adresse IP de l'utilisateur à Google. Remplacées par des polices locales via @fontsource, bundlées dans l'application.
🔧 Corrections techniques
🗄
Correction
createAppointment : RangeError SQLite corrigé
better-sqlite3 levait une RangeError: Missing named parameter lorsque des champs optionnels (guest_*, patient_id) étaient absents du binding object. Les paramètres SQL sont maintenant tous mappés explicitement avec ?? null.
⚙️
Correction
tsconfig : résolution des types partagés
TypeScript résolvait shared/types.ts depuis un fichier .d.ts compilé périmé plutôt que depuis les sources, causant des erreurs « membre non exporté » sur les nouveaux types. Correction : baseUrl ajouté dans tsconfig.node.json, references retirées du tsconfig renderer.
🗂
Correction
invoiceService : champs email/phone manquants
L'interface locale InvoiceData dans invoiceService.ts ne déclarait pas les champs email et phone que le code utilisait, causant une erreur de compilation bloquant le build.
