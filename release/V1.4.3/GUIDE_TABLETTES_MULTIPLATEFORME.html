Tablette Windows
Surface Pro, Surface Go, Lenovo Tab, Samsung Galaxy Tab S9 FE Windows…
⚡ Approche Electron — Application native (RECOMMANDÉE)
🟢 Effort : 3-4 semaines

L'application Synoria fonctionne déjà sur toute tablette Windows. Le travail consiste uniquement à optimiser l'interface pour le tactile — aucune modification de la base de données, de la sécurité ou de l'architecture.

✅ Avantages
  • Installateur .exe existant, aucune réécriture
  • 100% de la codebase réutilisée
  • SQLite + chiffrement identiques au desktop
  • RGPD et sécurité inchangés
  • Export Excel, PDF, impression natifs
  • Mise à jour automatique (electron-updater)
  • Panneau admin Ctrl+Shift+Alt+A fonctionnel
  • Clé USB portable conservée
  • Pas de commission store
❌ Inconvénients
  • Ne fonctionne pas sur Android ni iPad
  • Boutons trop petits pour les doigts (à corriger)
  • Pas de gestes swipe natifs
  • Hover states CSS à désactiver
  • Clavier virtuel peut masquer le formulaire
  • Formulaire séance long sur petit écran
📋 Marche à suivre
1
Activer la détection tactile en CSSAjouter @media (pointer: coarse) dans global.css pour cibler les écrans tactiles
2
Agrandir les zones cliquablesMinimum 44px de hauteur pour tous les boutons et liens (standard tactile)
3
Supprimer les effets hoverLes :hover restent actifs sur tactile → ajouter @media (hover: none) pour les désactiver
4
Adapter le formulaire séance (NewSessionPage)Sections pliables/dépliables sur tablette, navigation par onglets pour les 10 sections
5
Gérer le clavier virtuel WindowsDétecter visualViewport.height pour remonter les champs masqués
6
Tester sur Surface Pro / Surface Go réelTester toutes les pages avec uniquement les doigts, sans clavier/souris
🏆
Approche fortement recommandée

C'est la solution la plus rapide, la plus sûre et la moins coûteuse. Synoria sur tablette Windows = le même logiciel, la même sécurité, la même conformité RGPD. Idéal pour commencer l'extension mobile.

🔮 Approche Capacitor / PWA — Application web dans le navigateur
🟡 Effort : 2-3 mois (si migration complète)

Capacitor ne cible pas Windows nativement — sur PC/tablette Windows, cela revient à créer une PWA installable via Chrome ou Edge. L'intérêt est de partager la même base de code avec Android/iOS.

✅ Avantages
  • Codebase partagée Android + iPad + Windows
  • Installation sans droits admin (PWA)
  • Mise à jour automatique via Service Worker
❌ Inconvénients
  • Réécriture de toute la couche IPC/Electron
  • SQLite WASM moins performant que natif
  • Stockage navigateur (Edge/Chrome) efaçable
  • Accès fichiers limité (pas de showOpenDialog)
  • Electron est déjà parfait sur Windows
  • Migration très coûteuse pour un gain nul sur Windows
⚠️
Non recommandé pour Windows seul

Capacitor/PWA sur Windows n'apporte rien de plus qu'Electron. Envisageable uniquement si tu décides de migrer vers Capacitor pour toutes les plateformes en même temps.

Tablette Android
Samsung Galaxy Tab, Lenovo Tab, Xiaomi Pad… Android 11+ recommandé
Important : Electron ne fonctionne pas sur Android. C'est une application desktop (Windows/Mac/Linux uniquement). Sur Android, deux options : APK via Capacitor ou PWA via Chrome.
🔮 Approche Capacitor — APK natif (RECOMMANDÉE)
🟡 Effort : 2-3 mois

Capacitor génère un vrai fichier APK Android à partir du frontend React existant. Les APIs natives (SQLite, fichiers) remplacent les équivalents Electron.

✅ Avantages
  • Vrai APK distribué directement (.apk par email)
  • SQLite natif sur l'appareil (rapide, fiable)
  • Stockage non effaçable par l'utilisateur
  • Accès fichiers natif Android
  • Pas de Play Store requis → 0% commission
  • Données 100% locales, RGPD conservé
  • ~70% du code React réutilisable
  • Web Crypto API = même chiffrement AES-256
  • Codebase partagée avec l'iPad
❌ Inconvénients
  • IPC Electron → architecture Capacitor à réécrire
  • Android Studio requis pour build
  • Impression limitée (pas de Ctrl+P natif)
  • Mise à jour manuelle (nouvel APK à distribuer)
  • Fragmentation Android (tester sur plusieurs appareils)
  • Panneau admin raccourci à adapter
📋 Marche à suivre
1
Extraire le frontend React d'ElectronSéparer src/renderer/ comme app web autonome (Vite)
2
Remplacer better-sqlite3Installer @capacitor-community/sqlite — API quasi-identique
3
Remplacer Node.js cryptoWeb Crypto API (AES-256-GCM + PBKDF2) natif dans Chrome Android
4
Remplacer les appels IPCwindow.mtcApi.xxx() → services locaux directs (plus d'IPC nécessaire)
5
Gérer les fichiers@capacitor/filesystem pour sauvegardes, exports Excel, PDF
6
Adapter l'UI pour tablette AndroidCSS tactile (44px min), navigation adaptée, sections pliables
7
Builder l'APKnpx cap add android + Android Studio → fichier .apk signé
8
Distribuer directementEnvoyer le .apk par email → praticien active "Sources inconnues" une fois
🔒 RGPD : SQLite sur l'appareil Android = données locales = même modèle RGPD qu'aujourd'hui. Aucun HDS requis. Les données n'quittent jamais l'appareil.
🌐 Approche PWA — Dans Chrome Android
🟢 Effort : 1-2 mois

Chrome sur Android est le meilleur navigateur pour les PWA — meilleur support que Safari iOS. L'app s'installe sur l'écran d'accueil et ressemble à une vraie app.

✅ Avantages
  • Effort moindre (1-2 mois)
  • Aucun build natif requis
  • Mise à jour automatique (Service Worker)
  • Fonctionne aussi sur Windows + iPad
  • OPFS généreux sur Android Chrome
  • File System Access API disponible
❌ Inconvénients
  • SQLite WASM (moins performant)
  • OPFS potentiellement vidé si cache Chrome nettoyé
  • Pas de vrai APK (ne passe pas au Play Store)
  • Accès fichiers moins fluide que natif
  • Impression moins fiable
  • Nécessite connexion initiale (download app)
📋 Marche à suivre
1
Extraire le frontend ReactApp web autonome depuis src/renderer/
2
SQLite WASMIntégrer @sqlite.org/sqlite-wasm + OPFS pour le stockage
3
Web Crypto APIRemplacer Node.js crypto (API identique, native dans Chrome)
4
Service WorkerWorkbox pour cache offline et mises à jour automatiques
5
manifest.jsonIcône, nom, couleur thème pour installation "Add to Home Screen"
6
Héberger la PWAVercel (déjà configuré) ou serveur HTTPS — l'app se télécharge une fois
⚠️ Risque PWA Android : si l'utilisateur vide le cache Chrome ou désinstalle Chrome, les données locales (OPFS) peuvent être perdues. Recommander des sauvegardes régulières.

iPad / Apple
iPad Pro, iPad Air, iPad mini — iOS 17+ requis (Safari 17)
Contrainte Apple : Electron ne fonctionne pas sur iPad. De plus, sur iOS, tous les navigateurs (Chrome, Firefox…) utilisent le moteur WebKit d'Apple — ils sont tous aussi restrictifs que Safari. Le build iOS avec Capacitor nécessite obligatoirement un Mac avec Xcode.
🔮 Approche Capacitor — App native iOS
🔴 Effort : 3-4 mois

Capacitor génère une vraie application iOS (.ipa). Distribution possible via App Store ou en entreprise (TestFlight / distribution directe sans store).

✅ Avantages
  • Vraie app native iOS — expérience optimale
  • SQLite natif (fiable, rapide)
  • Stockage permanent (non effaçable)
  • Accès fichiers via Files app
  • AirPrint pour impression
  • Codebase partagée avec Android
  • TestFlight pour distribution bêta (sans App Store)
❌ Inconvénients
  • Mac + Xcode obligatoires pour builder
  • Apple Developer Program : 99 €/an
  • App Store médical : validation stricte (1-7 jours)
  • 30% de commission Apple sur ventes App Store
  • Certification médicale peut être refusée
  • Révision Apple à chaque mise à jour
  • Plateforme la plus contraignante
📋 Marche à suivre
1
Extraire le frontend (si pas déjà fait Android)App React autonome depuis src/renderer/
2
Intégrer Capacitor SQLite + FilesystemMême plugins que Android — build partagé
3
Ajouter la cible iOSnpx cap add ios — nécessite un Mac avec Xcode installé
4
Souscrire Apple Developer Program99 €/an sur developer.apple.com — obligatoire pour builder un IPA
5
Configurer les certificats et provisioning profilesDans Xcode : signing, bundle identifier, entitlements
6
Tester via TestFlightDistribution bêta jusqu'à 10 000 testeurs, sans App Store, validation légère
7
Soumettre sur App Store (optionnel)Catégorie "Médecin / Santé" — prévoir justificatifs d'usage professionnel
⚠️ Alternative sans App Store : Enterprise Distribution ou TestFlight permanent permettent de distribuer l'app iOS sans passer par le store, sans les 30% de commission. Idéal pour un logiciel B2B professionnel.
🌐 Approche PWA — Dans Safari iOS
🟡 Effort : 3-4 mois (même que Capacitor)

PWA installable via Safari sur iPad. Nécessite iOS 17+ pour OPFS et SQLite WASM. Techniquement faisable mais avec les contraintes les plus strictes de toutes les plateformes.

✅ Avantages
  • Pas d'App Store requis
  • Pas d'Apple Developer (0 €)
  • Pas de commission
  • Mise à jour automatique
  • Codebase identique Android PWA
❌ Inconvénients
  • iOS 17 minimum (iPad ancien exclu)
  • Quota stockage Apple très restrictif (~1 Go)
  • OPFS peut être vidé par iOS (gestion mémoire)
  • Accès fichiers quasi inexistant
  • Pas de vrai sélecteur de fichiers
  • "Add to Home Screen" manuel et peu intuitif
  • Impression AirPrint seulement
  • Safari WebKit seul autorisé sur iOS
📋 Marche à suivre
1
Même base que PWA AndroidCodebase partagée — React + SQLite WASM + Web Crypto
2
Tester spécifiquement sur Safari iOS 17OPFS et SQLite WASM se comportent différemment sur WebKit
3
Implémenter backup régulier obligatoireiOS peut vider l'OPFS → export JSON automatique conseillé
4
Guide utilisateur "Installer sur iPad"Safari → partager → "Sur l'écran d'accueil" — peu intuitif, doc nécessaire
5
Gérer les quotas de stockageDemander la permission d'accès au stockage persistant (storage.persist())
⛔ Point critique : Sur iPad, Safari iOS peut supprimer les données OPFS si l'iPad manque d'espace ou si l'app n'a pas été utilisée depuis 7 jours (comportement observé sur iOS). Risque de perte de données patient.

Recommandations RGPD par plateforme

🪟
Windows Electron Aucune modification RGPD requise. Exact même modèle que la version desktop actuelle.
🤖
Android Capacitor Adapter la notice Art.13 pour mentionner "stockage sur l'appareil Android du praticien". Ajouter une procédure de sauvegarde obligatoire dans le guide utilisateur.
🍎
iPad Capacitor Même chose qu'Android. Ajouter dans l'app un rappel de sauvegarde hebdomadaire automatique (iOS peut gérer agressivement la mémoire).
⚠️
iPad PWA — Risque à documenter Informer explicitement l'utilisateur que l'app fonctionne dans Safari et que vider le cache Safari = perte des données. Recommander la sauvegarde quotidienne. Ce risque est atypique pour un logiciel médical.

Recommandations & Roadmap
Ordre de priorité et planning suggéré

Verdict par plateforme

🥇
1er choix — Tablette Windows (Electron, 3-4 semaines)

Zéro réécriture, zéro risque, zéro changement RGPD. Toutes les fonctionnalités conservées. La Surface Pro ou un équivalent Windows offre exactement la même expérience que le PC, avec écran tactile. C'est le moyen le plus rapide d'offrir une solution tablette aux praticiens.

🥈
2e choix — Android Capacitor (2-3 mois)

La migration vers Capacitor est significative mais raisonnable. Le marché Android représente 72% des tablettes. Distribution directe par APK sans Play Store ni commission. Si tu fais Android, tu as 80% du travail déjà fait pour iPad ensuite.

🥉
3e choix — iPad Capacitor (3-4 mois, après Android)

Viable mais coûteux (Apple Developer 99€/an, Mac requis, validation Apple). À envisager si une partie de tes praticiens utilisent iPad Pro. TestFlight permet de contourner l'App Store pour un usage professionnel. Partage la codebase Android.

⚠️
Alternative légère — PWA Android (1-2 mois)

Solution intermédiaire si tu ne veux pas gérer les APK et Android Studio. Moins robuste que Capacitor (stockage navigateur) mais plus simple à déployer. Fonctionne bien pour des praticiens tech-savvy.

⛔
À éviter — iPad PWA Safari uniquement

Les restrictions Apple sur Safari iOS (quotas mémoire, règle des 7 jours, accès fichiers limité) en font la solution la plus risquée pour des données médicales. Le gain (pas d'Apple Developer) ne justifie pas le risque de perte de données.

Roadmap suggérée

1
Phase 1 — Windows tactile (Maintenant)
Adapter le CSS de l'app Electron existante pour les écrans tactiles. Tester sur Surface Pro. Distribuer un nouvel installateur V1.4.4 "tablette optimisée".
⏱ 3-4 semaines · 0 € coût additionnel
2
Phase 2 — Android Capacitor (V1.5 ou V2.0)
Extraire le frontend React, intégrer Capacitor SQLite et Filesystem, adapter l'UI tactile. Builder un APK et le distribuer directement aux bêta-testeurs Android.
⏱ 2-3 mois · Coût : Android Studio (gratuit), EAS Build optionnel
3
Phase 3 — iPad Capacitor (Si demande confirmée)
Ajouter la cible iOS à la codebase Capacitor Android (80% déjà fait). Souscrire Apple Developer. Builder via Mac (ou Mac cloud). Distribuer via TestFlight.
⏱ +1 mois (après Android) · Coût : 99 €/an Apple Developer + accès Mac

Ce qu'il ne faut jamais faire

⛔ À éviter absolument
  • Ajouter Supabase ou tout cloud → HDS obligatoire, RGPD refondu, coûts mensuels
  • Stocker les données dans localStorage (pas OPFS) → effaçable par l'utilisateur sans avertissement
  • Publier sur l'App Store Apple sans avoir vérifié les règles pour les apps médicales (risque de rejet)
  • Utiliser SQLite WASM sans stratégie de backup → risque de perte de données sur iOS
  • Exiger une connexion internet pour fonctionner → inadapté en cabinet
