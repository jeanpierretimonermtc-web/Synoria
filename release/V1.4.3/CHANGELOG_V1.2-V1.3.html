🏗️ Architecture générale

L'application passe d'un formulaire MTC unique en dur à une architecture modulaire : un Core commun (patients, calendrier, comptabilité, sécurité) et des Plugins définissant l'anamnèse spécifique à chaque spécialité.

SituationFormulaire affiché
Aucun plugin installéFormulaire générique simple (anamnèse, traitement, réactions)
Plugin MTC JP installéFormulaire MTC complet intégré (barrage, systèmes, énergie…)
Plugin Kinésiologie installéFormulaire kinésiologie personnalisé (Charlotte DECAENS)
  • 📐
    Moteur de rendu dynamique — PluginFormRenderer
    Lit un fichier plugin JSON et génère automatiquement le formulaire de séance. Supporte 12 types de champs : text, textarea, richtext, select, radio, checkbox, checkboxgroup, tags, rating, number, date, separator. Grille automatique (half / third / full) avec regroupement intelligent des champs.
  • 📄
    Format plugin JSON — standard ouvert
    Un fichier .plugin.json décrit sections, champs, labels, couleurs et types. Nouveau praticien = nouveau fichier JSON (3 à 5 jours de travail), sans modifier le core. La clé "useBuiltinForm": true active le formulaire MTC intégré complet.
  • ⚙️
    Gestion depuis les Paramètres
    Section "Plugin de spécialité" dans Paramètres : import d'un fichier .json, affichage du plugin actif (nom, version, sections), suppression pour revenir au formulaire générique. Validation complète du fichier à l'import avec messages d'erreur explicites.
  • 🌿
    Plugin MTC JP — MTC_JP.plugin.json
    Quand installé : affiche le formulaire MTC intégré complet (identique à la v1.1.0) incluant langue/pouls, questionnaire systèmes, tests énergétiques, barrage homéopathique. Distribué dans release/V1.2.0/Plugins/.
  • 🔬
    Plugin Kinésiologie — Kinesio_Charlotte.plugin.json
    Fidèle à la fiche papier de Charlotte DECAENS KINESIOLOGIE (SIRET 930727706). 5 sections : Contexte du RDV (objet, familial, professionnel, suivi médical), État psycho-émotionnel (stress 1-5, sommeil, émotions, vision avenir), Motivation & Objectifs, Suivi. Tous les champs en richtext pour la mise en forme.
  • 📊
    Résumé de séance adapté aux données plugin
    La page Résumé affiche les sections et champs du plugin utilisé lors de la séance. Le schéma complet du plugin est sauvegardé dans chaque séance (pluginSchema) — les résumés restent lisibles même si le plugin change par la suite.
  • 📥
    Export Excel incluant les données plugin
    La section "DONNÉES [NOM PLUGIN]" est ajoutée automatiquement à l'export Excel, avec sous-sections colorées par section de l'anamnèse.

📋 Formulaire générique simple (sans plugin)

Quand aucun plugin n'est installé, un formulaire universel s'affiche :

📝 Page Nouvelle séance — Mode générique
Section 3Anamnèse — Histoire & contexte du patient (richtext) + Observations cliniques (richtext)
Section 4Traitement effectué — Description (richtext) + Techniques utilisées (tags)
Section 5Résultats & Réactions — Richtext avec mise en forme
💡 Ces champs (anamnese, traitement_notes, reactions) sont les mêmes que dans le formulaire MTC — compatibles avec la base de données et les exports existants.

Améliorations post v1.1.0
Calendrier & Rendez-vous
  • ⏰
    Créneaux horaires à 5 minutes Nouveau
    Les sélecteurs d'heure dans la modal RDV proposent désormais des créneaux toutes les 5 minutes de 07h00 à 20h00 (contre 30 min auparavant). La vue planning du jour garde des blocs visuels de 30 min mais affiche les rendez-vous à leur heure précise. Même amélioration dans la section "Prochain RDV" de Nouvelle séance.
  • 👤
    Motif de venue pour nouveaux patients Nouveau
    Lors de la prise de RDV pour un patient sans fiche, le formulaire inclut désormais un champ "Motif de venue" à côté du téléphone. Le motif s'affiche sur le planning, dans le résumé mensuel et dans la liste des RDV. Badge "· nouveau" visible sur le créneau pour identifier les patients non-fichés.
  • 🔁
    Synchronisation bidirectionnelle RDV ↔ Séance (améliorations)
    Section "Prochain RDV" enrichie dans Nouvelle séance : heure début + heure fin + note. Les RDV existants du patient s'affichent en liste cliquable pour être liés à la séance (évite la duplication). Un RDV lié est mis à jour (pas dupliqué) à l'enregistrement.
  • 👥
    Création de fiche patient depuis un RDV invité
    Bouton "👤 Créer la fiche patient" dans la modal RDV : crée automatiquement la fiche avec nom/prénom/téléphone renseignés et lie le RDV au nouveau patient. Supprime les champs invité.

✍️ Barre de mise en forme

  • 🎨
    Barre globale unique — toutes les pages
    Une seule barre de formatage, placée sous la navigation, active sur toutes les zones de notes. Grisée quand aucune zone de texte n'est active. Raccourcis Ctrl+B, Ctrl+I, Ctrl+U maintenus. Bouton "✕ fmt" pour supprimer toute mise en forme d'un texte sélectionné.

🗂️ Migration base de données — v7

Deux nouvelles structures de données ajoutées automatiquement au premier lancement :

Champ / TableTypeRôle
patients.consent_givenINTEGER (0/1)Consentement RGPD donné ou non
patients.consent_dateTEXT (date)Date du consentement
access_log (table)—Journal horodaté de tous les accès aux fiches
  • ✅
    Consentement patient — sur chaque fiche Nouveau
    Case à cocher "Le patient a été informé et a donné son consentement" dans le formulaire patient. Si cochée, un champ date du consentement apparaît (pré-rempli avec la date du jour). La page RGPD affiche le taux de consentement global avec barre de progression et listes des patients avec/sans consentement.
  • 📋
    Journal d'accès minimal Nouveau
    Journalisation automatique et horodatée de chaque accès aux données : ouverture d'une fiche patient (Modifier), consultation d'un résumé de séance. Table access_log avec patient, action, détail et timestamp ISO. Consultable + filtrable dans la page RGPD onglet "Journal d'accès".
  • ⏰
    Durée de conservation configurable + alertes Nouveau
    Curseur dans ⚙️ Paramètres → RGPD pour définir la durée de conservation (1 à 30 ans, défaut 10 ans). Deux niveaux d'alerte dans la page RGPD : ⛔ Dépassé et ⚠️ Approchant. Basé sur la date de la dernière séance du patient.
  • 📄
    Notice d'information patient (Art. 13) Nouveau
    Texte par défaut conforme RGPD inclus, personnalisable dans Paramètres → RGPD. Visible et imprimable depuis la page RGPD onglet "Notice patient". Remise au patient pour l'informer du traitement de ses données.
  • 📊
    Registre des traitements (Art. 30 RGPD) Nouveau
    Export HTML complet généré automatiquement avec : responsable du traitement, finalité et base légale (Art. 9(2)(h)), catégories de données, destinataires, transferts hors UE (aucun), mesures de sécurité techniques (AES-256, PBKDF2, journal d'accès, localisation 100% locale), statistiques en temps réel (patients, séances, taux de consentement), droits des personnes concernées. Ouvre dans le navigateur — imprimable en PDF.
