Envoyer les informations de diagnostic

Synoria v1.4.3  ·  Guide d'assistance technique

📌 Quand envoyer un rapport de diagnostic ?

  • L'application plante ou se ferme de manière inattendue
  • Un bug reproductible affecte votre travail (données non sauvegardées, affichage incorrect...)
  • L'application refuse de démarrer ou reste bloquée au chargement
  • Une opération (sauvegarde, export, import) échoue avec un message d'erreur
  • Tout comportement anormal que le support vous a demandé d'investiguer

Méthode 1   Recommandé  — Rapport de diagnostic intégré

Cette méthode génère un rapport lisible qui compile automatiquement les informations techniques utiles au support (version, configuration système, dernières erreurs). Utilisez-la en priorité si Synoria démarre normalement.

1

Ouvrir Synoria et vous connecter avec votre mot de passe habituel.

2

Dans la barre de navigation, cliquer sur Paramètres, puis sur l'onglet Support.

3

Cliquer sur le bouton "Générer le rapport de diagnostic".

Synoria compile les informations techniques (version, système d'exploitation, journal d'erreurs récentes) et prépare un fichier texte.

4

Un fichier .txt s'ouvre automatiquement dans le Bloc-notes (ou votre éditeur de texte par défaut).

Vérifiez que le fichier contient bien des informations avant de le fermer.

5

Envoyer le fichier par email à support@synoria.fr en pièce jointe.

Décrivez dans le corps de l'email le problème observé et les étapes pour le reproduire (voir modèle ci-dessous).

Méthode 2   Si l'app ne démarre pas  — Fichier log brut

Si Synoria ne parvient pas à démarrer et que la méthode 1 est inaccessible, vous pouvez envoyer le fichier de log technique directement depuis l'Explorateur Windows.

1

Ouvrir l'Explorateur Windows (raccourci : touche Win + E).

2

Cliquer sur la barre d'adresse en haut de la fenêtre, taper le chemin suivant et appuyer sur Entrée :

%APPDATA%\Synoria\
Note : %APPDATA% est une variable système Windows qui pointe généralement vers C:\Users\[VotreNom]\AppData\Roaming\. Il n'est pas nécessaire de la remplacer manuellement — Windows la résout automatiquement.
3

Dans ce dossier, repérer le fichier synoria.log.

Si plusieurs fichiers log sont présents (ex. synoria.old.log), envoyez le fichier synoria.log en priorité, et l'ancien si le support vous le demande.

4

Envoyer le fichier synoria.log en pièce jointe à support@synoria.fr avec une description du problème rencontré (voir modèle ci-dessous).

Ce que doit contenir votre email au support

📝 Informations à inclure dans le corps de l'email

  • Description du problème : expliquez ce qui ne fonctionne pas, ce que vous voyez à l'écran (message d'erreur, comportement inattendu, etc.).
  • Étapes pour reproduire : listez les actions exactes qui déclenchent le problème (ex. : "1. J'ouvre la fiche d'un patient, 2. je clique sur Nouvelle séance, 3. j'enregistre → l'application se ferme").
  • Fréquence : le problème est-il systématique ou intermittent ?
  • Version de Windows : Windows 10 ou Windows 11, et si possible la version exacte (Settings → Système → À propos).
  • Fichier joint : le rapport de diagnostic .txt (méthode 1) ou le fichier synoria.log (méthode 2).
